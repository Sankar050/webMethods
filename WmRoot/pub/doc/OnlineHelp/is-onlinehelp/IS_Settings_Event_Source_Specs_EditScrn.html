Event Specification
Connection Alias Name
Streaming connection alias for the streaming server to which events that are instances of this specification will be sent. Once an event specification is created, the selected streaming connection alias cannot be changed.
Event Specification Name
A unique name for the event specification. Once an event specification is created, the specification name cannot be changed.
Topic
Topic Name
Topic on the streaming provider in which events are stored.
Key
Type
The data type for the event key. The selected type indicates how Integration Server encodes the key object in a published event to bytes and how Integration Server decodes the bytes in the key object of an event received by an event trigger.
none. A type of none indicates no data will be sent or received for the key object. For example, if you set the key type to "none" and the value type to "String", Integration Server will send or receive only the value object as String. Integration Server will not send or receive a key.
raw. A type of raw indicates that no decoding is performed for the key object by event triggers that receive the event. The event trigger service receives a byte array for the key. You might use "raw" if you want the trigger service to call a service that decodes the data or if you want the trigger service to simply forward the raw data onto another service. For example, suppose you want an event trigger to forward an event to another topic only if the key = "Canada". Because the trigger processing depends only on the key object, you do not need to decode the value object. You might select String as the type for key and raw as the type for value.

Decode as String
If you selected Double, Float, Integer or Long in the Type field, select the Decode as String check box if you want Integration Server to decode the byte array to a String. A String value may be easier to work with in Integration Server.
Document Type Name
If you selected JSON or XML in the Type field and you want to use a document type to direct encoding and decoding, type the fully qualified name of the IS document type, JSON document type, or XML document type in the Document Type Name field .
Charset Name
If you selected String, JSON, or XML in the Type field, type the charset name to use for encoding and decoding in the Charset Name field. The default is UTF-8.
Value
Type
The data type for the event value. The selected type indicates how Integration Server encodes the value object in a published event to bytes and how Integration Server decodes the bytes in the value object of an event received by an event trigger.
none. A type of none indicates no data will be sent or received for the value object. For example, if you set the key type to "String" and the value type to "none", Integration Server will send or receive only the key object as String. Integration Server will not send or receive a value.
raw. A type of raw indicates that no decoding is performed for the value object by event triggers that receive the event. The event trigger service receives a byte array for the value. You might use "raw" if you want the trigger service to call a service that decodes the data or if you want the trigger service to simply forward the raw data onto another service.
