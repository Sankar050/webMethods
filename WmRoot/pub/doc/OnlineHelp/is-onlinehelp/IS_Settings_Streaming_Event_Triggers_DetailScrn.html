General Settings
Trigger Name
Fully qualified name of the event trigger.
Package
Package in which the streaming connection alias is saved.
Connection Alias
Streaming connection alias used by the event trigger.
Client ID
Client ID for the event trigger. Integration Server creates a client ID for event triggers by appending the event trigger name to the client prefix. In Kafka, the client ID is known as the group ID.
Enabled
Indicates whether or not the streaming connection alias is enabled. When a streaming connection alias is enabled, Integration Server can use it to send and receive messages from the streaming provider server on behalf of publishing services and event triggers, respectively. A disabled streaming connection alias cannot be used to send messages to or receive messages.
Receive and Decode
streamName
Name for the stream.
Event Specification Name
Event specification that identifies the topic from which the trigger receives events.
Topic
Topic on the streaming provider from which the event trigger retrieves events.
Key Deserializer
Data type for the event key. The type indicates how Integration Server encodes the key object in a published event to bytes and how Integration Server decodes the bytes in the key object of an event received by an event trigger.
Value Deserializer
The data type for the event value. The type indicates how Integration Server encodes the value object in a published event to bytes and how Integration Server decodes the bytes in the value object of an event received by an event trigger.
Process
window
Indicates whether the event trigger processes a single event, a count-based window of events, or a time-based window of events A summary of window configuration follows.
streamName
Name of the stream.
streamProcessingSteps
Stream processing steps used in the event trigger, including summary of the step configuration.
Properties
Commit Mode
Determines when the consumer for an event trigger commits the events it received to the streaming provider.
Polling Request Size
Number of events each polling request retrieves.
Polling Interval
Frequency with which the consumers for the trigger poll the streaming provider for events.
On Disconnect
Determines how an event trigger handles received, unprocessed events upon notification of disconnect from the streaming provider.
Consumer Count Minimum
Minimum number of consumers to create for this event trigger.
Consumer Count Maximum
Maximum number of consumers to create for this event trigger.
Add Consumers After
The length of time to wait before creating another consumer.
Remove Inactive Consumers After
The length of time that a consumer can be inactive before Integration Server removes it.
Use Parallel Streams
Whether Integration Server splits intermediate processing across multiple threads.
Transient Error Handling
How Integration Server handles a transient error for an event trigger. A transient error occurs when a service executed as part of event processing throws an ISRuntimeException
Fatal Error Handling
How Integration Server responds when a fatal error occurs during event trigger processing.
Resource Monitoring Service
The service that Integration Server executes to determine the availability of resources associated with stream processing for the event trigger.
Runtime Configuration Parameters
Displays the configuration parameters and values that Integration Server uses for the consumer created for an event trigger.
