Edit messaging logger details
Use this page to change the settings for the messaging audit logger.
Note:
In Microservices Runtime, the Messaging Logger is disabled by default. Therefore, the logger is not available in the Logs menu of the navigation panel.
Audit Logger Configuration
Name
The name of this logger, Messaging Logger.
Enabled
Whether the logger is to write entries to the messaging audit log when enhanced logging is configured for a Universal Messaging connection alias of a JMS connection alias. The default is Yes.
Mode
Mode for writing log entries. The default is Synchronous.
Option
Description
Synchronous
The logger writes log entries directly to the destination.
You might use synchronous logging when you have an application that requires some type of auditing to succeed, and you do not want to proceed without knowing that the auditing occurred.
If synchronous logging for an entry fails (for example, because the destination is unavailable), the logger switches to asynchronous logging for that entry (see below).
Asynchronous
The logger writes log entries to a queue, and the queue later writes the entries to the destination.
Guaranteed
Whether Integration Server maintains the internal queue in memory or on disk when the logger is logging asynchronously. The default is No, which indicates the queue is not guaranteed.
Option
Description
No
Integration Server maintains the queue in memory. This option provides better logging performance. However, if Integration Server shuts down abnormally, the log entries in the queue will be lost.
Yes
Integration Server persists the queue on disk. This option is safer but can adversely affect logging performance.
Destination
Destination for log entries. The messaging logger always writes to a file and cannot be configured to write to a database.
Maximum Queue Size
For asynchronous logging, maximum number of log entries the queue can hold. The default is 100000. Specify numerals only; for example, do not include commas or periods.
Choose a value that accommodates your system's average volume for log entries. If your logging volume has sudden spikes, the queue can usually catch up by writing the pending entries during lulls. Make sure the Integration Server host machine has enough disk space or memory to accommodate the largest possible size of the queue as specified in this field, as well as the requirements of other applications the Integration Server is hosting. If the logger is writing to a database, the queue's insertion of logged data into the database is constrained by the database's availability and connections limit.
If the queue reaches its maximum, the logger writes the log entries to a file called FailedAudit_yyyymmdd_hhmmss.log in the Integration Server_directory/instances/instance_name/logs directory. You can scan the file to find events that are not logged.
Queue Provider
The messaging logger uses the internal, light-weight queue as the audit logging queue. The messaging logger cannot be configured to use a queue on Universal Messaging as the audit logging queue. The messaging audit logger always uses the internal queue as the audit logging queue.
Connection Alias
This field does not apply to the messaging audit logger because the messaging audit logger cannot be configured to use a queue Universal Messaging as the audit logging queue. The messaging audit logger always uses the internal queue as the audit logging queue.
Maximum Retries
This field does not apply to the messaging audit logger because the messaging audit logger cannot be configured to write to a database.
Wait Between Retries
This field does not apply to the messaging audit logger because the messaging audit logger cannot be configured to write to a database.
Audit Logger Reader Configuration
Number of Readers
This field does not apply to the messaging audit logger because the messaging audit logger cannot be configured to use a queue Universal Messaging as the audit logging queue. The messaging audit logger always uses the internal queue as the audit logging queue.
Messaging Audit Log Level
Logging Level
The debugging level at which the messaging logger writes data to the messaging audit log. Each logging level includes the indicated type of messages plus all messages from the levels above it (for example, the Info level includes Fatal, Error, Warn, and Info messages). The debugging level determines how much detail the message audit logger records in the messaging audit log.
Note:
The logging level for the messaging audit logger can be different than the logging level for the 0168 Messaging (Enhanced Logging) server log facility for the server log.
The logging level can be one of the following:
Level
Description
Info
Main sending and receiving actions such as success or failure of sent or received messages.
Debug
Details of sending and receiving activities such as preprocessing and message acknowledgment.
Trace
Further details of sending and reeving activities, such as the start and end of polling the messaging provider for additional messages.
Note:
Trace-level messages are provided for JMS messaging only.
